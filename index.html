    🎚 Cutoff 기준 정의

    각 타입별 임계값을 자유롭게 조정

    ROI별 PASS 조건은 두 가지를 모두 만족해야 합니다 — ① usability 점수 평균이 임계값 이상이고 ② 임계 점수(N) 이하로 받은 환자 비율이 X% 이하. 예: Type A는 "평균 4점 이상 + 3점 이하 받은 환자가 10% 이하"일 때 PASS.

    💡 GTV 권장: marginal miss (FN) 임상 위험이 over-contour (FP)보다 큽니다. Detection rate가 낮은 모델은 Type A(엄격)에서 자동 fail되도록 평균 점수 기준을 보수적으로 잡는 것을 권장합니다. 구조적인 sensitivity/FP rate cutoff은 향후 추가 예정.

    Type 평균 ≥ 문제 점수 (≤) 문제 비율 (≤) 해석

    GTV 3-DIMENSION CUTOFF (Detection + Quality + Sub-metric AND)

    GTV PASS는 세 차원을 모두 통과해야 합니다 — Detection(검출률), Quality(점수, 위 표), Sub-metric(체크 비율). Quality는 위에서 설정. 여기는 Detection + Sub-metric의 GTV-specific threshold입니다. 현재는 heuristic default — User study 후 refinement 예정 (Spec §8).

    Type Sens ≥ FP rate ≤ severe miss ≤ sig miss ≤ major incl ≤ sig artifact ≤
